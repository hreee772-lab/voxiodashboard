Data Security We implement industry-standard technical and organizational measures to protect your data: ● Encryption in Transit: All data transmitted between your browser and our servers is encrypted using TLS 1.2 or higher. HTTPS is enforced for all connections with HSTS (HTTP Strict Transport Security) enabled with a one-year policy. ● Encryption at Rest: Connected app tokens are encrypted using AWS KMS. Database storage is encrypted through AWS RDS native encryption. Files stored in S3 use server-side encryption. ● Credential Security: OAuth tokens for connected apps are encrypted before database storage using AWS KMS in production environments and are only decrypted at the moment of use. ● Account Isolation: All data is scoped to individual user accounts. Database queries are filtered by authenticated user identity. Other users cannot access your data. ● Access Controls: Internal access to production systems is restricted. Authentication uses Firebase Auth with Google OAuth for secure sign-in. ● Security Headers: Content Security Policy (CSP), X-Frame-Options, X-Content-Type-Options, and other security headers are enforced across all responses. No system is 100% secure. While we take reasonable measures to protect your data, we cannot guarantee absolute security against all threats. 7.
